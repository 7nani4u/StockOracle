# StockOracle 패턴 엔진 리팩터링 기록

## 기존 구조 요약

- 기존 캔들 엔진은 단일·2봉·3봉 조합을 최신 봉 기준으로 판정한다. 캔들 패턴은 삭제하지 않고 `candlestick_context` 보조 근거로 유지한다.
- 기존 기하 엔진은 최근 고점 3개와 저점 3개를 별도로 비교해 삼각형·쐐기·이중 천장/바닥을 판정했다. 피벗의 교차 순서, 확인 시점, 상태, 넥라인, 무효화, 목표가가 없었으므로 공통 엔진으로 교체했다.
- 새 엔진은 피벗·ATR·거래량을 한 번 계산하고 모든 구조 탐지에서 재사용한다.
- 기존 `ChartPatternAnalyzer(df).detect_patterns()`와 `find_local_extrema()` 호출 형식은 호환 래퍼로 유지한다.

## 최종 런타임 패턴 레지스트리

실제 OHLC 구조 탐지와 목표가를 제공하는 패턴은 다음과 같다.

1. `inverse_head_shoulders`
2. `head_shoulders_top`
3. `double_bottom`
4. `double_top`
5. `triple_bottom`
6. `triple_top`
7. `symmetrical_triangle` — 방향 확정 전에는 표시/점수 제외
8. `ascending_triangle`
9. `descending_triangle`
10. `falling_wedge`
11. `rising_wedge`
12. `bullish_consolidation` — 깃발·페넌트·직사각형 공통 상승 탐지
13. `bearish_consolidation` — 깃발·페넌트·직사각형 공통 하락 탐지

기존 캔들 패턴은 호환성과 기존 분석 문맥을 위해 유지하지만, 기하 구조와 같은 패턴으로 중복 합산하지 않는다. 형성 중·돌파 대기 패턴은 `score_eligible=false`이고 확정 구조만 점수·목표가 신뢰도 보강에 사용한다.

## 상태와 룩어헤드 방지

- 피벗은 오른쪽 `pivot_window`개 봉이 지난 시점에만 확정된다.
- 각 E 좌표에는 원래 봉 `index`와 실제 사용 가능 시점 `confirmed_index`를 함께 저장한다.
- `forming`: E1~E5 일부만 완성. 다음 피벗 종류·예상 가격 범위를 반환한다.
- `awaiting_breakout`: 구조는 완성됐지만 종가·ATR 버퍼·확인 봉 조건이 미충족이다.
- `confirmed`: 구조 마지막 피벗이 확정된 뒤 종가 기준 돌파가 유지됐다.
- `invalidated`: 구조 완성 또는 돌파 뒤 무효화선/실패 리테스트 조건을 종가로 위반했다.
- `expired`: 설정된 형성 유효 봉 동안 돌파되지 않았다.

탐지 결과와 차트 오버레이는 동일한 `points`, `neckline`, `target` 객체를 사용하며 시각화용 좌표를 다시 계산하지 않는다.

## 목표가 및 TP1~TP5 통합

- H&S: 헤드에서 넥라인까지 높이를 확인선 기준으로 투영한다.
- 이중·삼중 반전: 평균 극점과 넥라인의 높이를 투영한다.
- 삼각형·쐐기: 돌파 시점의 패턴 폭을 투영한다.
- 공식이 없는 특성 계열은 목표가를 생성하지 않는다.
- 패턴 목표는 기존 TP 가격을 덮어쓰지 않는다. 가장 가까운 TP가 0.75 ATR 이내일 때 `sources`에 근거를 병합하고, 확정·완성도 70점 이상이면 가능성 범위를 2%p만 보강한다.
- 멀리 떨어진 목표는 `pattern_target_integration.rejected`에 이유와 함께 남긴다.
- 형성 중 목표는 `provisional=true`이며 확률을 높이지 않는다.

## 호환성 주의사항

- 화면과 API가 지원하는 분석 기간은 `1일·3일·1주·2주·1개월·6개월·1년·2년·5년`으로 통일한다.
- `calc_risk()`의 기존 인자는 그대로이며 마지막 선택 인자 `chart_patterns`만 추가됐다.
- 기존 TP1~TP5 개수·가격 정렬·시나리오 간 비중복 출력은 유지된다.
- API 응답에는 `chart_patterns`, `chart_data.pattern_overlays`, `risk_scenarios.pattern_target_integration`이 추가된다.
- 차트는 확정 패턴을 실선과 `CONF`, 형성 중/돌파 대기를 점선과 `FORM`/`WAIT`로 구분한다.
- 외부 API와 신규 라이브러리는 추가하지 않았다.
